## 4. Fix — Architecture Description (Section 3.2.1)

**What the proposal says:**
- 3 layers: Collection → AI Processing (4 tiers) → SIEM Forwarding

**What's missing:**
The actual project has a **dual-classifier architecture** with source-type routing:

```
event ──► source-type router
              │
              ├─ syslog / web / firewall / winevent ──► syslog classifier (XGBoost, 100 feat)
              │                                              │
              │                                              └─► final score (no tier-2)
              │
              └─ generic / other ──► HDFS classifier (XGBoost, 2255 feat)
                                         │
                                         ├─ p < 0.10 ──► trust tier-1 (confident benign)
                                         ├─ p > 0.90 ──► trust tier-1 (confident failure)
                                         └─ 0.10–0.90 ──► tier-2 (SecureBERT2.0 ONNX)
```

**Fix:** Add this diagram and explanation after the 4-tier description in section 3.2.1. This is an important innovation the proposal misses.

Also add that the **4 tiers** in the actual implementation are:
- Tier 1a: Sigma Rule Engine (fast pattern match, runs on every event)
- Tier 1b: Dual XGBoost classifiers (syslog vs HDFS, source-type routed)
- Tier 1c: SecureBERT2.0 Tier-2 ONNX (only for uncertain HDFS scores)
- Tier 2: BiEncoder embedding + FAISS (dedup + ATT&CK retrieval)
- Tier 3: NER extraction + CrossEncoder re-ranking
- Final: Composite weighted score
